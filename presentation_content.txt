COREFLEX PILATES STUDIO - PRESENTATION CONTENT

===============================================================================
SLIDE 1: PROJECT OVERVIEW
===============================================================================
Title: Coreflex Pilates Studio Management System

- Next.js 15 application for Pilates studio management
- Role-based authentication (Client/Admin)
- Real-time session booking and management
- Built with TypeScript, MongoDB, and Tailwind CSS
- OTP-based secure authentication system

===============================================================================
SLIDE 2: HOME PAGE (/)
===============================================================================
Title: Landing Page - First Impression
File: app/page.tsx

Purpose: Public-facing homepage with studio information

Key Features:
- Hero section with studio background image
- Contact information display (WhatsApp, Phone, Email, Location)
- Navigation to login and contact sections
- Responsive design with Tailwind CSS animations
- Clean, professional studio branding

===============================================================================
SLIDE 3: LOGIN PAGE (/login)
===============================================================================
Title: Authentication Gateway
File: app/login/page.tsx

Purpose: OTP-based authentication system

Key Features:
- Email input with OTP generation
- OTP verification with countdown timer
- JWT token generation for session management
- Automatic redirect based on user role (admin/client)
- Secure, passwordless authentication

===============================================================================
SLIDE 4: CLIENT DASHBOARD (/dashboard)
===============================================================================
Title: Client Dashboard - Main Hub
File: app/dashboard/page.tsx

Purpose: Central hub for client activities

Key Features:
- Welcome message with personalized greeting

- Active packages display with session counts
- Expired packages section
- Package expiry dates and session tracking
- User-friendly interface with clear navigation

===============================================================================
SLIDE 5: BOOK SESSIONS (/dashboard/book)
===============================================================================
Title: Session Booking Interface
File: app/dashboard/book/page.tsx
Components: SlotCardsContainer.tsx, SlotCard.tsx

Purpose: Interactive session booking system

Key Features:
- Date selector with 15-day booking window
- Program-wise slot organization (Morning/Evening)
- Real-time availability checking
- Color-coded slot status:
  * Green: Available
  * Yellow: Filling Fast
  * Orange: Last Spot
  * Red: Full
  * Blue: Already Booked
- One-click booking with confirmation

SLIDE 10: API ARCHITECTURE
===============================================================================
Title: Backend Integration

Key API Endpoints:
- /api/auth/request-otp - OTP generation
- /api/auth/verify-otp - OTP verification
- /api/slot - Session management
- /api/user - User management
- /api/program - Program management

Technical Features:
- JWT-based authentication with role verification
- MongoDB database with Mongoose ODM
- Real-time data updates
- IST timezone management
- RESTful API design
